# Documentation Localization Policy

Kyberion supports localization, but the documentation system must retain a single canonical source of truth.

## Canonical Rule

Official architecture, governance, and contract documents should be authored in English first.
English is the canonical source unless a document explicitly states otherwise.

## Localization Rule

Localized documentation is allowed when it materially improves operator usability, onboarding, or communication quality.
However, localized variants should be treated as `derived operator aids`, not as the primary contract source.

## Recommended Structure

Use this pattern:

```text
docs/
  <canonical-doc>.md
  i18n/
    ja/
      <canonical-doc>.md
```

For knowledge documents:

```text
knowledge/public/
  ...
  i18n/
    ja/
      ...
```

## What Should Usually Stay Canonical

These documents should usually remain English-first:

- architecture
- governance
- schemas
- system contracts
- operating policies
- actuator contracts

## What Is Good to Localize

These are good candidates for localization:

- operator instructions
- onboarding aids
- quick-reference guides
- UI labels
- status summaries
- clarification prompts

## Vocabulary Rule

UI- and operator-facing wording should not be localized by scattering translated strings across surfaces.
Instead, surfaces should prefer governed catalogs such as:

- [`knowledge/public/orchestration/user-facing-vocabulary.json`](/Users/famao/kyberion/knowledge/public/orchestration/user-facing-vocabulary.json)

This rule applies mainly to `fixed labels`, `section headers`, and short status words.
It does not require every free-form explanation or clarification sentence to be pre-translated.
Those may be generated by the LLM from canonical English-oriented contracts.

## Drift Rule

If a localized document diverges from the canonical English source, the English source wins.
Localized copies should be updated as soon as practical.

## Practical Guidance

Use English for:

- contracts
- schemas
- architecture
- governance

Use localization for:

- presentation
- conversation
- operator experience

This keeps the system globally understandable while still making local operation comfortable.
