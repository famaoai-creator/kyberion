# Multi-Platform Development Standard v1.0

この文書は、Kyberion エコシステムにおいて、複数の OS (macOS, Windows, Linux) に対応したスキルおよび基盤機能を開発するための標準を定義する。

## 1. 物理的な抽象化 (Platform Abstraction Layer)

直接 `os.platform()` をチェックしたり、OS 固有のシェルコマンド（`say`, `screencapture` 等）を呼び出すことは禁止される。代わりに `@agent/core` の `platform` ユーティリティを使用しなければならない。

### 使用例:
```typescript
import { platform } from '@agent/core';

// 画面キャプチャ
await platform.captureScreen('output.jpg');

// 音声合成
await platform.speak('Hello World');

// アプリケーション/URLを開く
await platform.open('https://google.com');
```

## 2. スキルの互換性宣言 (Skill Compatibility)

特定の OS でしか動作しないスキルは、`SKILL.md` の Frontmatter で明示的に制限しなければならない。

```yaml
---
name: macOS-only-skill
platforms: [darwin]
---
```

## 3. ガバナンスとヘルスチェック

1.  **自動検知**: `npm run generate-index` は各スキルの `platforms` メタデータを抽出し、グローバルインデックスに記録する。
2.  **実行時ガード**: CLI はスキル実行前に現在のホスト OS とスキルの互換性をチェックし、不一致の場合は実行を拒否する。
3.  **ヘルスチェック**: `scripts/check_skills_health.ts` は、現在の環境で動作しないスキルを `🚫 UNSUPPORTED` としてフラグを立てる。
