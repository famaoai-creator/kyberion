# 検討報告：マルチテナント・ナレッジガバナンスの最適化案
**日付:** 2026-03-04  
**コンテキスト:** 8社兼務のCEOという特殊状況下での、コンテキスト・ノイズの排除、FISC/ISMS準拠、および安全な知恵の共有（Sharing）に関するアーキテクチャの検討。

## 1. 背景と目的
CEOが8つの組織・プロジェクトを同時に統治する際、情報の混線を防ぎ、AIエージェント（LLM）に対して「その時必要な最小限の情報」のみを提示したい。また、情報の出所（所有者）を明確にし、他者への共有時には適切なフィルタリング（ホスピタリティ）を施す仕組みを、管理コストを抑えつつ実現する。

## 2. 検討案の比較マトリックス

| 案 | コンセプト | メリット | デメリット | 整合性・適合性 |
| :--- | :--- | :--- | :--- | :--- |
| **1. 物理ディレクトリ分離** | `knowledge/confidential/{tenant_id}/` にフォルダを分ける | 直感的で、`grep` 等の標準ツールで探しやすく管理が容易。 | 階層が深くなる。共用情報の扱いで物理的なコピー（重複）が発生する。 | **低**: 重複不整合のリスク。 |
| **2. ADF 可視性タグ** | ファイル内に `visibility: corp` 等のメタデータを書き込む | 1つのファイルで、見る人の立場に合わせて内容を「伸縮」させられる。 | 全ファイルにタグを書く手間。AIによるパース漏れの懸念。 | **中**: 柔軟だが運用負荷大。 |
| **3. Vault Mount (外部)** | 会社のデータは別リポジトリに置き、実行時のみ `./context/` へマウント | セキュリティ強度が最高。エンジン（AI）がステートレスになり、漏洩リスク最小。 | ミッションごとに「金庫」を開ける手間。OS側のリンク管理が複雑。 | **高**: 物理隔離としては最強。 |
| **4. Contextual Lens** | ミッションフォルダ内に必要なファイルだけリンクを集約 | LLMのコンテキストを物理的に最小化。他テナントのノイズをゼロにできる。 | 同名ファイル（`rules.md`等）が別テナントにあると、リンク時に衝突する。 | **中**: 効率的だが名前空間に弱点。 |
| **5. Namespace Mirroring** | `vault/{id}/` (読専) と `wisdom/{id}/` (書専) を ID でペア管理 | 組織階層から解放されたフラットな管理。論理的に美しい。 | ミッションごとに「どのIDをマウントするか」を常に意識する必要がある。 | **高**: 論理的整合性は高い。 |
| **6. Sovereign Funnel** | `inbox/` に放り込めば AI が自動で分類・ハッシュ管理 | CEOの管理コストがゼロ。重複も物理的に発生しない。 | **FISC/ISMS的に説明責任が果たせない**（「なぜここにあるか」がAI任せ）。 | **低**: 個人用には良いが組織にはNG。 |
| **7. Hybrid Ledger (最終案)** | **「厳格な登録コマンド」＋「動的な名前空間マウント」** | FISC準拠の証跡（台帳）を残しつつ、重複排除とコンテキスト絞り込みを両立。 | 最初の一回だけ `ingest` コマンドを打つ手間（配置や台帳登録は自動）。 | **◎**: 統治と利便性の最高地点。 |

## 3. 議論の要点

### 3.1 名前空間 vs 物理配置
別テナントで同名ファイル（`rules.md` 等）が存在するリスクを考慮すると、単なる集約ではなく、**テナントをプレフィックスとした名前空間**（例：`a_corp/rules.md`）としてAIに見せることが、情報の出所を誤認させないために不可欠。

### 3.2 ISMSと資産トレーサビリティ
UIがどうあれ、システムの本質は**「情報資産台帳（Ledger）」**であるべき。誰が、いつ、何の目的で持ち込み、誰に見せて良いのかを台帳で一元管理することが、エンタープライズ級のガバナンスの基盤となる。

### 3.3 共有（おもてなし）の作法
知恵をAさんに共有するとは、単にファイルを送ることではない。以下の「再現キット」をパッケージ化することである：
1. **Logic**: 実行手順（YAML）
2. **Background**: 必要な前提知識（Public）
3. **Anonymized Data**: 機密を隠したサンプルデータ（Anonymized Private）

## 4. 現在の結論
「案7：Hybrid Sovereign Ledger」を最有力候補とする。人間が所有権を宣言（監査対応）し、AIがそれを元にコンテキストをフィルタリング（効率化）するハイブリッドな構成が、8社兼務のCEOにとって最も整合的である。
