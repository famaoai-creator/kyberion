# OpenClaw Architecture and Conceptual Analysis

OpenClaw (https://github.com/openclaw/openclaw) の調査結果と、Kyberion エコシステムとの比較分析。

## 1. OpenClaw の核心コンセプト
OpenClaw は、ユーザーが日常的に使用するメッセージングアプリ（WhatsApp, Telegram, Slack, iMessage等）をインターフェースとし、ローカルデバイス上で「実際に仕事をする」パーソナル AI アシスタントである。

### 主要な特徴
- **Omni-Channel**: ユーザーとの接点を最大化し、チャットから直接システムを操作する。
- **Infrastructure First**: 実行バイナリの Allowlist、パスの安全性、Heartbeat 監視など、実行環境の堅牢性を重視している。
- **Local Native**: macOS の `say` (TTS) や Apple 系アプリ（Notes, Reminders）との深い統合。

## 2. Kyberion との比較分析

### 共通点
- **Skill-based Modularity**: 機能を `skills/` 単位でカプセル化し、拡張する。
- **Developer-Centric**: TypeScript/Node.js を基盤とした、拡張の容易なアーキテクチャ。

### 差異
| 項目 | OpenClaw | Kyberion |
| :--- | :--- | :--- |
| **アイデンティティ** | 個人アシスタント。 | 主権者の自律的な代理人（エンジニアリング・経営）。 |
| **実行モデル** | イベント・リアクティブ型。 | ミッション・タスク駆動型（確定的な証跡）。 |
| **統治** | バイナリレベルの制限。 | 3-Tier ナレッジティア分離と Sovereign Shield。 |
| **視覚・音声** | 標準的な TTS/STT。 | 10フレーム視覚バッファ ＆ 非同期音声合成。 |

## 3. Kyberion が取り入れるべき知見
1. **バイナリ Allowlist**: 現在の `platform.ts` に、実行を許可するコマンドのパターンマッチングを追加することで、セキュリティをさらに強化できる。
2. **Heartbeat & Diagnostics**: エージェントの健康状態を常時監視する仕組みを、現在の `doctor.ts` から常駐デーモンへと進化させる余地がある。
3. **App Integration**: macOS 標準アプリとの連携スキルを増やすことで、日常的なタスクの自動化（カレンダー連携等）を強化できる。
