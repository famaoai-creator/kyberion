# Data Harvesting Best Practices: URL Resolution & Secure Download

## Overview
When autonomous agents (e.g., `browser-navigator`) extract links from a DOM snapshot, those links (`href` attributes) are often relative paths. Agents must systematically resolve these into absolute URLs and use governed tools for ingestion, rather than executing arbitrary shell commands.

## 1. Contextual URL Resolution
The `snapshot.json` generated by the browser always includes the current page context:
```json
{
  "url": "https://www.fsa.go.jp/news/r7/sonota/20260303/aidp.html",
  "elements": [
    { "ref_id": 26, "type": "a", "href": "/news/r7/sonota/20260303/aidp_version1.1.pdf" }
  ]
}
```
**Rule:** An agent MUST evaluate if an `href` starts with `http`. If it starts with `/`, the agent MUST prepend the origin (e.g., `https://www.fsa.go.jp`) extracted from the root `url` property before attempting any interaction or download.

## 2. Secure Ingestion to Vault
Direct shell commands like `curl` or `wget` should be avoided for downloading binary assets into `vault/`. They bypass the ecosystem's audit logging and proxy configurations.

**Rule:** Use dedicated ingestion skills (e.g., `api-fetcher` or a specific downloader script) that utilize `@agent/core/secure-io` to write to `vault/`. This ensures the action is tracked in the `TASK_BOARD.md` and `contract.json`.

## 3. Separation of Discovery and Analysis
- **Discovery**: Use `browser-navigator` (Omni-Browser) to output a clean `snapshot.json`.
- **Harvesting**: Use jq or script (`scripts/url_resolver.ts`) to extract and construct the absolute URLs.
- **Ingestion**: Download the target file.
- **Analysis**: Use `doc-to-text` on the downloaded artifact and stream the output to a safe location (`scratch/` or `knowledge/`)—never overwriting the original artifact in `vault/`.
