# Deal Tracker — プロダクトロードマップ

## 背景・課題

顧客（中小企業・営業部門5名）のヒアリングで判明した根本課題：

- Excelで案件管理しているが、人によって書き方がバラバラ
- フォローアップ期日を頭と手帳で管理しており、**忘れが月数件発生** → 商談機会損失
- 管理者（部長）が案件全体を俯瞰できる手段がない

## ゴール

営業担当者が「朝開いたら今日すべきことがわかる」＋管理者が「全体の進捗を一目で把握できる」ツール。

---

## Phase 1 — MVP（今回実装）

**対象:** 社内5名 + 管理者1名  
**期間:** 即日リリース

| 機能 | 説明 |
|---|---|
| 案件登録 | 会社名・担当者・金額・ステージ・フォローアップ日 |
| パイプラインビュー | 4ステージ（初回接触/提案/交渉/受注） のカンバン |
| 今日のフォロー | 期日が今日以前の案件を赤くハイライト |
| 案件編集・削除 | インライン編集 |
| データ永続化 | localStorage（ブラウザ内、ゼロサーバー） |

---

## Phase 2 — チーム共有（〜1ヶ月後）

| 機能 | 説明 |
|---|---|
| バックエンドAPI | Node.js + SQLiteで共有データベース化 |
| ユーザー認証 | 担当者ごとのログイン |
| 担当者フィルター | 自分の案件だけ / 全件 切り替え |
| コメント機能 | 案件に活動履歴を記録 |

---

## Phase 3 — モバイル＆通知（〜3ヶ月後）

| 機能 | 説明 |
|---|---|
| PWA対応 | スマホからも使えるようにする |
| プッシュ通知 | フォローアップ期日の前日リマインド |
| 管理者ダッシュボード | ファネル分析・転換率・売上予測 |
| CSV エクスポート | 既存Excelレポートへの橋渡し |

---

## 技術方針

- **Phase 1:** 純粋な HTML + CSS + JavaScript（ゼロ依存、即デプロイ）
- **Phase 2:** Express + SQLite + 既存Kyberionコアライブラリ活用
- **Phase 3:** PWA + Web Push API

## KPI

| 指標 | Phase 1目標 |
|---|---|
| フォローアップ忘れ件数 | 月3件 → 0件 |
| 案件登録率 | 全商談の100%をシステム管理 |
| 管理者確認時間 | 週次Excel集計30分 → リアルタイム0分 |
